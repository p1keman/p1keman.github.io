在跟踪exec方法执行命令时，发现的一点问题

exec方法实质上也是调用的processBuilder类的start方法来执行命令。

使用exec执行命令时，需要调用StringTokenizer类来对参数进行分割，

最后会把这个数组传给create方法

第一个空格前的会被当作模块传入 ，第一个空格后的都会被当作参数传入，不会被当作模块

之前的payload是echo xxx>test.txt echo会被当作一个模块，但是Windows下根本没有这个模块，他只是cmd里面的一个命令

所以可以把cmd作为模块，命令作为参数 改成这样cmd /c echo xxx>test.txt
